# セキュリティポリシー

JPKI Local Signer は、マイナンバーカードの署名用電子証明書と秘密鍵を扱うアプリです。本書は脅威モデルの概要と、脆弱性を発見した場合の報告方法を説明します。

## 脅威モデルの概要

### 秘密鍵はカードの外に出ない

署名鍵はマイナンバーカードのICチップ内にのみ存在し、いかなる場合もカードの外へ取り出されません。署名処理は `COMPUTE DIGITAL SIGNATURE` コマンドによりカード内で実行され、アプリが受け取るのは署名値（RSA署名バイト列）のみです。アプリ自身が秘密鍵を扱う・保持することは設計上ありません。

### 機微データを永続化・ログ出力しない

以下のデータは、メモリ上で処理に必要な期間のみ保持し、**ディスクへの永続化・ログ出力のいずれも行いません**。

- 署名用PIN（暗証番号）
- ハッシュ値・DigestInfo
- カードから受け取った署名値
- カードからのAPDU応答（証明書本体を除く生の応答データ）

クラッシュレポート収集SDK等のサードパーティ計測系SDKも導入していません（意図せぬ経路での機微データ漏洩を避けるため）。

### ネットワーク通信を一切行わない

本アプリは `URLSession` をはじめとするいかなる通信APIもソースコード中にリンクしていません。署名対象のPDF・証明書・PIN・署名値が端末外（サーバー、クラウド、第三者事業者）へ送信されることは設計上ありません。

この主張は次のスクリプトで機械的に検証できます（CIでも実行可能です）。

```sh
./scripts/check-no-network.sh
```

`Sources/` および `JPKILocalSignerApp/` 配下を対象に、`URLSession` / `URLRequest` / `NWConnection` / `Network` フレームワークの `import` / `CFSocket` / `getaddrinfo` / `NSURLConnection` などのネットワーキング関連シンボルの有無を検査し、検出した場合は非ゼロ終了コードで失敗します。

### PINロックのリスクと緩和策

マイナンバーカードの署名用PINは、**5回連続で照合に失敗するとロックされ**、以後は市区町村窓口での初期化（再設定）が必要になります。これはカード仕様上の制約であり、本アプリでは解除できません。

本アプリでは以下の方法でロックのリスクを緩和しています。

- PIN照合（VERIFY）を実行する前に、残り試行回数を取得する専用のAPDU照会を行います。この照会自体は試行回数を消費しません。
- 残り試行回数が少ない場合は、照合を実行する前に利用者へ警告を表示します。
- ロックに至る可能性がある操作の前に、失敗時の影響（窓口での初期化が必要になること）を明示します。

## 署名検証機能について

本アプリは署名済みPDFの**静的検証**を端末内で行います（各署名の ByteRange 整合・
CMS パース・messageDigest 一致・RSA 署名値の検証）。証明書の信頼分類は、
**アプリに同梱した JPKI 署名用CA ルート証明書（J-LIS 公表値と照合済み）への
チェーン連鎖をオフラインで暗号検証**して判定します（JPKI / 自己署名 / 他認証局 /
検証不能）。ルート証明書のフィンガープリントはアプリの「このアプリについて」画面と
[`Sources/SelfVerify/JPKITrustAnchors.swift`](Sources/SelfVerify/JPKITrustAnchors.swift)
に明示しており、誰でも J-LIS 公表値と再照合できます。この検証は補助であり、
正式な有効性判定を代替しません（下記スコープ外）。

## 対象外の脅威（スコープ外）

- **失効確認（CRL/OCSP等）は行いません。** 署名用電子証明書が失効・期限切れであるかどうかの検証は、提出先（登記・供託オンライン申請システム等）の署名検証者側の責務です。ネットワーク通信を行わない方針（NFR-01）とも整合します。
- **信頼リストの自動更新は行いません。** 同梱ルートは明示的なコード更新でのみ変わります（更新時は J-LIS 公表値との照合を必須とします）。
- **登記システムでの受理可否の保証はしません。** 生成したPDFが提出先システムで受理されるかどうかは、提出者自身が事前に確認する必要があります。アプリ内「登記適合チェック」は形式要件の目安です。

## 制度上の位置づけに関する重要な注意

本アプリは電子署名を **生成** するツールであり、署名を **検証** するサービスではありません。公的個人認証法第17条に基づく主務大臣認定は「署名検証者」に対する制度であり、署名生成のみを行う本アプリはその対象外です。本アプリの利用は自己責任でお願いします。

## 脆弱性の報告方法

セキュリティ上の懸念（機微データの意図しない漏洩経路、ネットワーク到達手段の混入、暗号処理の実装不備など）を発見した場合は、**公開のGitHub Issueには投稿せず**、以下のいずれかの方法で非公開に報告してください。

- 本リポジトリの GitHub の **Security** タブから **Security Advisory（プライベート報告）** を作成する。
- それが利用できない場合は、リポジトリのメンテナーへ直接連絡する手段（メール等）を確保している場合はそちらを利用する。

報告には可能な範囲で以下を含めてください。

- 再現手順（可能であれば最小構成）
- 想定される影響範囲（例: 機微データの漏洩経路、署名の偽造可能性など）
- 該当するファイル・関数（把握している場合）

対応可能なコミュニティベースのプロジェクトであるため、対応にはお時間をいただく場合があります。あらかじめご了承ください。
